# 2026-04-25 工作日志

## admin-config 工程配置保存逻辑修复

**问题根因**：`bulk_save_projects` 先 `ProjectCatalog.objects.all().delete()` 全清再 `bulk_create`，但 `HoboRequirementLedger` 有 `ForeignKey` 到 `ProjectCatalog` 且 `on_delete=PROTECT`，导致删除时触发 IntegrityError，但错误信息被错误地描述为"重复工程编码"。

**修复**：改为 upsert 逻辑——先查数据库已有记录，对比 payload：更新已有记录、新增不存在记录、删除 payload 中不存在的记录。不再使用全量 delete。
- 验证 payload 内无重复编码（同原来）
- 用 `bulk_update` 批量更新已有记录
- 用 `bulk_create` 批量新增新记录
- 用 `filter(...__in=...).delete()` 删除多余的记录
- 错误信息改为泛化 `f"保存失败：{e}"` 便于调试
